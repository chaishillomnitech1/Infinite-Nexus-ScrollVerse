## â“ FAQ

### General Questions

**Q: Do I need blockchain/Web3 experience?**  
A: Not for most contributions! Many areas (docs, UI, tests) don't require Web3 knowledge.

**Q: How long does PR review take?**  
A: Usually 2-7 days. Complex changes may take longer.

**Q: Can I work on multiple issues?**  
A: Start with one. Once merged, you can take on more.

**Q: What if my PR isn't perfect?**  
A: That's okay! Reviewers will help improve it.

### Technical Questions

**Q: Why is there no build step?**  
A: We're a static site. HTML/CSS/JS run directly in the browser.

**Q: Can I add a new dependency?**  
A: Discuss it first in an issue. We keep dependencies minimal.

**Q: How do I test Web3 features?**  
A: Use testnets (Goerli, Mumbai) with test wallets. Never use real funds.

**Q: What's the 528Hz frequency?**  
A: The "Miracle Tone" - a Solfeggio frequency used for healing and transformation. We use it as a design principle throughout the project.

### Contribution Questions

**Q: How do I get assigned to an issue?**  
A: Comment on the issue expressing interest. Maintainers will assign you.

**Q: Can I create my own issues?**  
A: Yes! Use the appropriate issue template.

**Q: What happens after my PR is merged?**  
A: You'll be credited as a contributor! You can then take on more issues.
